Critical: Deduct AFTER Success

Credits are only deducted after a successful API response. If the AI request fails, the user keeps their credits. This prevents charging for failed requests.

‚öõÔ∏è Atomic Deduction 

-- PostgreSQL RPC Function
CREATE FUNCTION deduct_credits(
  p_user_id UUID,
  p_amount INTEGER
) RETURNS BOOLEAN AS $$
BEGIN
  -- Lock row for update
  SELECT balance FROM credits
  WHERE user_id = p_user_id
  FOR UPDATE;
  
  -- Check & deduct atomically
  UPDATE credits
  SET balance = balance - p_amount
  WHERE user_id = p_user_id
    AND balance >= p_amount;
    
  RETURN FOUND;
END;
$$ LANGUAGE plpgsql;

Scaling Strategies

1
Connection Pooling

Supabase uses PgBouncer for connection pooling. Thousands of concurrent users share a pool of database connections.

2
Indexed Queries

All frequently queried columns (user_id, created_at) have indexes. O(log n) lookup time regardless of table size.

3
RPC Functions

Credit operations run as stored procedures. Single round-trip instead of multiple queries. Reduces latency and load.

4
Horizontal Scaling

Supabase Pro/Enterprise supports read replicas. Scale reads infinitely while writes go to primary.
